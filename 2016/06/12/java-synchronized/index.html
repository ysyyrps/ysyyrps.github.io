java synchronized详解

java

锁

互斥锁

  • 给某个对象加上互斥锁之后，那么这个对象在任何时刻，只能有一个线程访问该对象。假如有多个线程都想访问该对象，那么只能等待正在访问该对象的那个线程访问完毕。

  • 场景：同时穿上衣和穿裤子

  • 结果：

  • 互斥

  • 场景：同时穿上衣和穿鞋子
  • 结果：互斥

  • 场景：同时吃苹果和吃梨
  • 结果：互斥

  • 场景：同时吃香蕉和吃西瓜

  • 结果：不互斥

  • 场景：peter穿鞋子和zuoci吃西瓜
  • 结果：不互斥

总结

  • static+synchronized修饰方法和synchronized(A.class)锁的是静态方法，即只能允许有一个进程访问这个类的静态代码段，而且静态方法间互斥访问
  • synchronized修饰方法和synchronized(this)锁的是对象，即同个对象互斥访问这些方法，不同个对象则不互斥
  • 锁静态方法和锁对象之间没有关联，并不互斥
