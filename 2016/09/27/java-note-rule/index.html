java注释规则

java

核心目的

  • 更加清晰的明白每个变量、每行代码、每个包具体的作用

    通用原则

  • 所有公开的服务接口，都【必须】有相应的注释说明；
  • 代码文件加入copyright声明
  • 注释内容要清楚、明了、含义明确、防止二义性；
  • 每次更新代码，【必须】同步更新相应注释；
  • 注释语言视项目而定，默认【建议】使用英文；
  • 建议在IDE里中的注释配置相关设置，创建人【必须】填写

    包注释

  • 在当前路径包添加package-info.java文件，在这个文件的注释里描述这个包类文件的作用，内容，以及作者等等
  • 具体例子

类（接口）注释

  • 描述此文件的主要内容及用处、作者等
  • @author 作者 @since 从什么时候开始的
  • 具体例子

方法外注释

  • 必须写上方法的作用，@Param写上参数的意义，@return 表名返回值是什么，@throws注明抛出的异常类型和什么时候抛出
  • 具体例子

方法内注释

  • 表示底下代码的作用
  • 具体例子

变量注释

  • 表名变量的意义、功能等
