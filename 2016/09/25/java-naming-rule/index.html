java命名规则

java

核心目的

  • 见名思义，提高代码的可读性和可维护性

    通用原则

  • 保持命名的一致性
  • 使用合适的单词表达
  • 只允许使用定义在缩略语列表中的缩略语或者大众熟知的缩略语，禁止中文拼音缩写
  • 禁止使用字母和数字以外的字符，包括”-“
  • 谨慎使用数字进行命名

    包命名

  • 比如com.google.aa.bb中com.google是google公司域名从后往前取，aa指项目名，bb则指的是模块名（有时候aa会省略掉，直接到模块bb）
  • 所有的包名必须使用英文小写
  • 尽量使用一个单词，如果有多个单词，则单词间禁止有其他字符

    类（接口）命名

  • 类名必须是名词，并遵循如下原则：[形容词+]名词(单数)，例如：AbstractRuleActionExecution；
  • 接口名遵循如下原则：[副词+]形容词，或者[形容词+]名词（单数）
  • 类（接口）名首字母必须大写，如多个单词组合，则每个单词首字母大写；
  • 异常类名要以Exception来结尾
  • 实现类名：为与接口进行区别要以Impl来结尾
  • 抽象类名：要以Abstract来开头

    变量命名

  • 变量名遵循如下格式：[形容词+ ]名词，例如：domainName；
  • 首字母小写，如多个单词组合，后续单词首字母大写，例如：inputFile；
  • 如果变量类型是一个集合类，则变量名中的单词必须是复数，例如：parameters；
  • 禁止使用java数据类型作为变量名前缀，例如：strName和StringName,建议使用有意义的单词（允许使用类型作为后缀）；

    方法命名

  • 包括以下的命名原则：
    1、[副词+]动词
    2、动词+名词
    3、is+形容词
    4、can+动词
    5、has+动词过去式
    6、contains+名词

  • 首字母【必须】小写，如多个单词组合，后续单词首字母大写；

常量命名

  • 常量名遵循如下格式：[形容词+ ]名词；
  • 所有单词都是大写，且单词间必须用‘_’连接；
  • 如果常量是集合类型，则常量名中的名词必须是复数；
  • 禁止使用java数据类型作为常量名前缀；
  • 所有常量都必须声明为final static;
