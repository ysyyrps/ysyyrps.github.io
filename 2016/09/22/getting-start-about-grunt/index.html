Grunt入门（前端自动化）

前端

Grunt

特点及用处

  • Grunt是一个基于JavaScript上的一个任务管理器
  • 通过它，可以用JavaScript去执行一些任务，比如验证各种HTML、CSS、JavaScript语法，压缩代码
  • Grunt用于自动化构建、测试、生成文档的项目管理工具

    入门指南

    安装NodeJs&npm

  • Grunt和Grunt的插件是依赖于NodeJs和npm。所以我们要先安装nodeJs，再安装npm(npm实际上是NodeJs的包管理器），具体安装此处略。
  • Grunt要求Node.js版本高于0.8.0
  • 安装Grunt前，先更新下npm,执行npm update -g npm

    安装Grunt-cli

  • 在命令行执行npm install -g grunt-cli，这个命令会将grunt命令行设置到你的系统路径下，允许你在任何目录执行它

    CLI如何工作的

  • 每次运行grunt的时候，它会通过nodeJs中require()指令系统去寻找本地安装的Grunt。因此你可以在你项目的任意子文件夹执行grunt命令
  • 如果本地安装的grunt被找到，那么CLI就会载入本地安装的库，从gruntfile获取配置，并执行。

    处理已存在的Grunt项目

  • 保证Grunt CLI已经安装、并且项目已经配置了package.json和Gruntfile
  1. 目录切换到你项目的根目录
  2. 用=npm install安装项目的依赖（安装node_modules）
  3. 执行grunt
  • 这些就是我们要做的所有的了。你也可以执行grunt –help列出安装grunt的任务列表，但一般来说，更推荐有一个项目的文档。
  • 注： npm install 加上–save-dev标志时，在package.json中,插件的版本号前会增加一个波浪号，表示如果插件有更新补丁，它会自动升级,例

开始一个新的grunt项目

  • 首先我们要给项目增加两个文件package.json和Gruntfile
  • package.json 这个文件是用来配置项目需要的npm modules、插件以及在npm商店中的版本号

  • 模板

  • Gruntfile 这个文件用来定义我们构建的任务和载入grunt插件，它名字可以是Gruntfile.js，也可以是Gruntfile.coffee。
